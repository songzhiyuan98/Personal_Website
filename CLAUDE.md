# Claude Code 项目协作规则

## 语言
- 默认使用 **简体中文** 回复。
- 即使我输入英文描述，也请用中文回答。

## 修改流程
1. **预案先行**：在生成任何代码 patch 之前，必须先给出：
   - 修改目标说明
   - 涉及的文件清单
   - 改动点解释
   - 可能风险与回滚方案

2. **分次改动**：
   - 如果涉及多个文件，请拆分成多次提交。
   - 每次仅改动 1–3 个文件，逐步给出 diff。

3. **审核确认**：
   - 在展示 patch 之前，必须先征求我的确认。
   - 我说“确认”之后，才可以生成并展示 patch。

## 代码风格
- 所有注释、解释性文字统一用中文。
- TypeScript 代码必须带类型定义。
- 动画交互优先使用 Framer Motion，避免直接写原始 DOM 操作。
